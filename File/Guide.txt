LINUX APPLICATION MANAGER – SETUP GUIDE
----------------------------------------
----------------------------------------

PROJECT OVERVIEW
------------------

This project demonstrates a simple Linux automation process where an administrator can install or remove packages across multiple servers using SSH.
Before running the automation script, some prerequisites and configurations must be completed on the admin machine and target servers.

---

1. CHECK PACKAGE MANAGER AND REQUIRED PACKAGES

First verify available package managers and required packages.

Example:

Check if apt/java/jre exists

```
apt
```

Install a package using yum

```
yum install java
```

---

2. RESOLVE CENTOS REPOSITORY ERROR

If you are using CentOS 7, you may encounter repository errors because official mirrors are deprecated.
To resolve this issue, update the repository configuration to use the CentOS Vault.

Run the following commands:

```
sudo sed -i 's|mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=os|#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=os|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|#baseurl=http://mirror.centos.org/centos/$releasever/os/$basearch/|baseurl=http://vault.centos.org/7.9.2009/os/$basearch/|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=updates|#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=updates|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|#baseurl=http://mirror.centos.org/centos/$releasever/updates/$basearch/|baseurl=http://vault.centos.org/7.9.2009/updates/$basearch/|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=extras|#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=extras|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|#baseurl=http://mirror.centos.org/centos/$releasever/extras/$basearch/|baseurl=http://vault.centos.org/7.9.2009/extras/$basearch/|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=centosplus|#mirrorlist=http://mirrorlist.centos.org/?release=$releasever&arch=$basearch&repo=centosplus|g' /etc/yum.repos.d/CentOS-Base.repo

sudo sed -i 's|#baseurl=http://mirror.centos.org/centos/$releasever/centosplus/$basearch/|baseurl=http://vault.centos.org/7.9.2009/centosplus/$basearch/|g' /etc/yum.repos.d/CentOS-Base.repo
```

Clean repository cache:

```
sudo yum clean all
```

Rebuild repository cache:

```
sudo yum makecache
```

---

3. INSTALL REQUIRED SSH UTILITIES

On the admin machine install SSH client tools.

```
sudo yum install openssh-clients -y
```

Verify SSH installation:

```
ssh -V
```

---

4. CONFIGURE PASSWORDLESS SSH LOGIN

To allow automation scripts to connect to servers without entering a password, configure SSH key authentication.

---

STEP 1 – GENERATE SSH KEY

Run the following command on the admin machine:

```
ssh-keygen
```

Press Enter to accept default settings.

This will create the following files:

```
~/.ssh/id_rsa
~/.ssh/id_rsa.pub
```

---

STEP 2 – COPY SSH KEY TO SERVERS

Copy the public key to each server.

Example:

```
ssh-copy-id user@192.168.1.10
```

Repeat this step for all servers:

```
ssh-copy-id user@192.168.1.25
ssh-copy-id user@192.168.1.26
```

---

STEP 3 – TEST PASSWORDLESS LOGIN

Verify that SSH login works without a password.

Example:

```
ssh user@192.168.1.26
```

If login occurs without asking for a password, configuration is successful.

---

Create a file on the admin machine that contains all target servers.

Command:

vim servers.txt
